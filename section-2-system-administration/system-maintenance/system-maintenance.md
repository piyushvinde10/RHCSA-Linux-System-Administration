# 🛠️ System Maintenance Commands in Linux

System maintenance commands are used to **shutdown, restart, or stop** a Linux system safely.

---

# 📂 Commands Overview

These commands help administrators manage system power operations.

---

# 1. Shutdown Command

To view manual:

man shutdown

To shutdown system:

shutdown

### Used for:
- Safely power off the system  
- Schedule shutdown  
- Notify logged-in users  

---

# 2. Reboot Command

To view manual:

man reboot

To restart system:

sudo reboot

### Used for:
- Restarting the system  
- Applying updates or changes  
- Fixing temporary issues  

---

# 3. Halt Command

To view manual:

man halt

To stop system:

sudo halt

### Used for:
- Stopping all CPU functions  
- Powering off the system (based on configuration)  

---

# 🔧 Important Notes

- These commands usually require **sudo (admin access)**  
- Always ensure no critical process is running before executing  
- Useful in both servers and personal systems  

---

# 🧰 Useful Tip

Use manual pages to understand commands:

man command_name

Example:

man shutdown

---

# 📊 Summary

| Command  | Purpose              |
|----------|----------------------|
| shutdown | Shutdown system      |
| reboot   | Restart system       |
| halt     | Stop system          |

---

# 🎯 Key Learning

- Helps in safe system management  
- Prevents data loss during shutdown  
- Essential for Linux administrators
